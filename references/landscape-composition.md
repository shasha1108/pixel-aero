# 风景画纵深构图 SOP —— 可复用公式与结构

> 本文件从"山谷里的风"创作迭代中提炼。**不绑定任何具体元素**（风车/房子/蝴蝶），只保留可跨风景场景迁移的公式。
>
> **触发条件**：场景类型为开放风景（田园/山谷/草原/湖畔/海滨）— 即 STEP 1 场景模式判定为"开放场景"且用户描述含自然地形+天空+远景。
>
> **通用原则**（屏幕像素锚定 / 对抗式检查 / 第一性原理流程）已上提到 `design-principles.md §十三~§十五`，全场景通用，本文件不再重复。
>
> 不适用：封闭容器（穹顶/球体/玻璃瓶）、纯室内、纯水下。

## 一、纵深骨架公式（7 层结构）

任何风景画都可以套用这个骨架。**层数不关键，层间色差才关键。**

```
L0  天空 (CSS 渐变 + N层云 + 暖金光源于左上角)
     ↓ 大气雾带（蓝灰半透明叠加）
L1  极远山脊（去饱和 55-65%，蓝通道 > 绿通道，几乎融入天空）
     ↓ 第二雾带
L2  远丘 + 极小主体元素（去饱和 35-45%，蓝移）
     ↓ 河流 / 道路（透视收窄，作为视线引导线）
L3  中远地形 + 中等主体元素 + 尺度参照建筑群（去饱和 15-25%）
     ↓ 路径网络（连接各建筑，近宽远窄）
L4  中近草地 + 大号主体元素 + 前景细节（正常饱和度）
     ↓
L5  近景草坡 + 特大主体元素 + 生物细节（微暖色增强）
     ↓
L6  画面底框 / 窗台
```

**每层必须回答三个问题**：
1. 这一层有什么**主体元素**？（大小、数量、颜色）
2. 这一层有什么**引导线**把视线拉到下一层？（路/河/栅栏/地形褶皱）
3. 这一层有什么**细节**让观众发现？（花草/生物/光影）

**如果某一层三个问题都答不上来 → 这一层是空壳，删掉它。**

## 二、爆款三要素（风景画验证表）

交付前逐条对照。每一项的答案必须是具体的实现方式，不能是"做了"。

| 要素 | 含义 | 验证问题 | 反模式 |
|------|------|---------|--------|
| **可进入的纵深** | 不止"远处有东西"，而是有路/河/引导线把眼睛拉进去 | 观众的眼睛从近景出发，能沿着什么路径走到远景？这条路在每一层都看得见吗？ | 路/河只画了一段就断了；引导线宽度不变（没有透视收窄） |
| **尺度参照物** | 小建筑/小人物 → 主体元素显得巨大 → 画面有史诗感 | 如果把主体元素旁边的参照物拿掉，主体元素还"大"吗？参照物是否在不同的深度层都有分布？ | 只有一栋建筑，且孤零零放在画面正中；参照物和主体元素在同一层，没有纵深分布 |
| **活的细节** | 生物/花粉/流水/摇曳的植物 → 不是静态壁纸，是一个世界 | 观众盯着画面10秒后，能不能发现至少一种"会动的东西"？这种东西在第一秒看得见吗（还是藏太深）？ | 粒子均匀散布无结构（AI痕迹）；生物飞行轨迹是平滑Perlin漂移而非分段转向；细节只在最前景一层 |

**检验标准**：找3个没看过设计稿的人，让他们看画面5秒后回答——"你觉得你能走进这个画面吗？""画面里最大的东西大概有多高（相对于什么）？""有什么东西让你觉得这个世界是活的？" 3个人里有2个答不出来 = 三要素没落地。

## 三、大气透视公式

远层和近层的颜色差异，不能靠"感觉调一调"——必须有量化的色差。

### 3.1 颜色偏移规则

```
远层 (L1):  R通道降低 30-40%, G通道降低 20-30%, B通道增加 10-20%, 透明度 0.5-0.7
中远 (L2):  R通道降低 15-25%, G通道降低 10-20%, B通道增加 5-10%,  透明度 0.65-0.8
中层 (L3):  R通道降低 5-10%,  G通道降低 5-10%,  B通道持平,         透明度 0.75-0.85
近层 (L5):  正常色，全饱和，无透明
```

### 3.2 色差检验公式

```
色差 = |R1-R2| + |G1-G2| + |B1-B2|
```

- **色差 < 30**：两层看起来像同一层 → 失败，必须增大偏移
- **色差 30-60**：可接受，但需要引导线辅助分层
- **色差 > 60**：清晰分层，肉眼可辨

### 3.3 防扁平清单

- [ ] L1 远层的蓝通道是否 **明显大于** 绿通道？（B > G + 10 以上）
- [ ] L5 近层的绿通道是否 **明显大于** 蓝通道？（G > B + 30 以上）
- [ ] L1 和 L5 的色差是否 > 80？（最远 vs 最近）
- [ ] 每层透明度是否随距离递减？

## 四、元素放置交叉验证（风景画专属）

任何放在地面上的元素，必须先验证它脚下的地形和周围的水体。

### 4.1 交叉验证流程

```
对于每个放置在地形上的元素 E(x, y)：
  1. 确定 E 所在的深度层 L
  2. 计算 L 层地形在 x 处的实际 Y 坐标（使用 terrainY 函数）
  3. 如果场景中有水体（河流/湖泊）：计算水体在 y 处的边界 [left, right]
  4. 验证：E.x 不在水体边界内
  5. 验证：E.y 与 terrainY(x) 的偏差在 ±3px 以内
```

### 4.2 验证代码模板

```javascript
function validatePlacement(elem, layerBaseY, layerAmp, layerFreq) {
  const terrainY = getTerrainY(elem.x, layerBaseY, layerAmp, layerFreq);
  const issues = [];
  
  if (abs(elem.y - terrainY) > 3) {
    issues.push(`Y mismatch: elem at ${elem.y}, terrain at ${terrainY}`);
  }
  
  const river = getRiverBounds(elem.y);
  if (river) {
    const riverLeft = river.cx - river.w / 2;
    const riverRight = river.cx + river.w / 2;
    if (elem.x > riverLeft && elem.x < riverRight) {
      issues.push(`Element in water: x=${elem.x}, river=[${riverLeft}, ${riverRight}]`);
    }
  }
  
  return issues;
}
```
